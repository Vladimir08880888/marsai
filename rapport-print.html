1. Objectif du commit

La page d'accueil du site Mars.AI affiche actuellement des donnees statiques (titres de films, chiffres du festival codes en dur dans le JSX). L'objectif de ce commit est de creer deux endpoints API publics cote backend pour fournir ces donnees dynamiquement depuis la base de donnees MySQL :

  • GET /home/featured — retourne les films en vedette (statut "selected" ou "finalist")
  • GET /home/stats — retourne les statistiques du festival (nombre de films, utilisateurs, etc.)

2. HomeController.js — Le controleur

Chemin : back/src/controllers/HomeController.js (37 lignes)

2.1 Imports

import Film from "../models/Video.js"; import User from "../models/User.js"; import { Op } from "sequelize";
  • Film — modele Sequelize defini dans Video.js, mappe la table films en base.
  • User — modele Sequelize de la table users, utilise pour la jointure (recuperer le nom du realisateur).
  • Op — operateurs Sequelize. On utilise Op.in pour filtrer par plusieurs valeurs.

2.2 Fonction getFeaturedVideos

function getFeaturedVideos(req, res) { Film.findAll({ where: { status: { [Op.in]: ["selected", "finalist"] } }, limit: 6, order: [["created_at", "DESC"]], include: [ { model: User, as: "user", attributes: ["id", "first_name", "last_name"], }, ], }).then((films) => { res.json(films); }).catch((error) => { res.status(500).json({ error: "Failed to fetch featured films" }); }); }

Analyse ligne par ligne :

ElementExplication
Film.findAll({...})Requete SELECT sur la table films. Retourne un tableau d'objets.
where: { status: { [Op.in]: [...] } }Clause SQL : WHERE status IN ('selected', 'finalist'). Filtre les films qui ont ete selectionnes ou qui sont finalistes.
limit: 6Maximum 6 resultats. La page d'accueil n'affiche que 6 films en vedette.
order: [["created_at", "DESC"]]Tri par date de creation decroissante : les films les plus recents apparaissent en premier.
include: [{ model: User, as: "user" }]Jointure LEFT JOIN avec la table users via la cle etrangere user_id. L'alias "user" correspond a l'association definie dans le modele Video.js.
attributes: ["id", "first_name", "last_name"]On ne retourne que 3 champs de l'utilisateur. Le mot de passe et l'email sont exclus pour des raisons de securite.
.then((films) => res.json(films))En cas de succes, retourne le tableau JSON avec code HTTP 200.
.catch((error) => res.status(500)...)En cas d'erreur (DB inaccessible, requete invalide), retourne un code 500 avec un message d'erreur.

Requete SQL generee :

SELECT films.*, users.id, users.first_name, users.last_name FROM films LEFT JOIN users ON films.user_id = users.id WHERE films.status IN ('selected', 'finalist') ORDER BY films.created_at DESC LIMIT 6;

Exemple de reponse :

[ { "id": 3, "title": "Neural Odyssey", "status": "finalist", "thumbnail": "uploads/images/neural.jpg", "youtube_link": "https://youtube.com/watch?v=...", "created_at": "2026-02-01T00:00:00.000Z", "user": { "id": 5, "first_name": "Jean", "last_name": "Dupont" } } ]

2.3 Fonction getStats

function getStats(req, res) { Promise.all([ Film.count(), User.count(), Film.count({ where: { status: "selected" } }), Film.count({ where: { status: "finalist" } }), ]).then(([total_films, total_users, total_selected, total_finalists]) => { res.json({ total_films, total_users, total_selected, total_finalists }); }).catch((error) => { res.status(500).json({ error: "Failed to fetch stats" }); }); }

Analyse ligne par ligne :

ElementExplication
Promise.all([...])Execute les 4 requetes COUNT en parallele. Chaque Film.count() retourne une Promise. Promise.all attend que les 4 soient resolues.
Film.count()SQL : SELECT COUNT(*) FROM films. Retourne le nombre total de films.
User.count()SQL : SELECT COUNT(*) FROM users. Retourne le nombre total d'utilisateurs.
Film.count({ where: { status: "selected" } })SQL : SELECT COUNT(*) FROM films WHERE status = 'selected'.
([total_films, total_users, ...])Destructuring : les 4 resultats du Promise.all sont assignes a des variables nommees.
res.json({ total_films, total_users, ... })Shorthand ES6 : equivalent a { total_films: total_films, ... }.
Pourquoi Promise.all ? Sans Promise.all, les 4 requetes seraient executees l'une apres l'autre (sequentiellement). Si chaque COUNT prend ~50ms, le total serait ~200ms. Avec Promise.all, elles s'executent en parallele et le total est ~50ms. C'est 4 fois plus rapide.

Exemple de reponse :

{ "total_films": 24, "total_users": 156, "total_selected": 8, "total_finalists": 3 }

2.4 Export

export default { getFeaturedVideos, getStats };

Export par defaut d'un objet contenant les deux fonctions. Cet objet est importe dans le routeur sous le nom HomeController.

3. Home.route.js — Le routeur

Chemin : back/src/routes/Home.route.js (10 lignes)

import express from "express"; import HomeController from "../controllers/HomeController.js"; const homeRouter = express.Router(); homeRouter.get("/featured", HomeController.getFeaturedVideos); homeRouter.get("/stats", HomeController.getStats); export default homeRouter;

Analyse :

  • express.Router() — cree un mini-routeur Express independant, qui sera monte sur un prefixe dans le routeur principal.
  • Deux routes GET declarees. Pas de middleware d'authentification car ces endpoints sont publics (accessibles sans connexion).
  • Quand ce routeur est monte sur /home, les URLs finales deviennent /home/featured et /home/stats.
